Graph generation (2011?)

Originators: S. Hartke    (presented by S. Hartke - REGS 2012)

Motivation: To gather experimental evidence for conjectures, we frequently would like to verify that all graphs in a given class satisfy a desired property. Current computational power limits the size and number of such graphs that we can check. We would thus like algorithms that are efficient in practice to generate graphs in given classes. Since the properties we are interested in are usually isomorph-invariant, we would like to perform isomorph-free generation: only one representative of each isomorphism class is generated.

Background: McKay [M] has a general method (called canonical deletion) of isomorph-free generation of combinatorial objects. He has implemented this algorithm for garphs in the program geng, part of his nauty package. Meringer [M2] gave a method for generating regular graphs using orderly generation. The algorithm is implemented in GENREG. Brinkmann [B] gave a fast method for generating cubic graphs, implemented as minibaum. Brinkmann, Goedgebeur, and McKay [BGM] have a faster method for cubic graphs, implemented as snarkhunter.

Problem: For a given degree sequence d that is graphic, find an (efficient) algorithm to generate all of the nonisomorphic realizations of d.

Comments: There has been recent progress in sampling uniformly at random from regular graphs; these methods (such as [MW], [SW], and [JS]) may be relevant.

Could a method for generating all the realizations of a degree sequence use 2-switches instead of building from smaller graphs? Nonregular sequences may have few enough realizations (and possible 2-switches) that it may be possible to keep track of switches.
